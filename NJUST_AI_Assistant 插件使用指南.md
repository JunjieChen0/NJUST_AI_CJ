# NJUST_AI_CJ 插件使用指南

本指南详细介绍 NJUST_AI_CJ 插件的各项功能及操作方法，帮助您充分利用这个强大的编程助手。😊

---

## 1. 插件安装与初始化

### 安装方法

1. 下载插件的 `.vsix` 文件（如 `njust-cj-2026.3.18.vsix`）
2. 在 VS Code 中，点击左侧扩展图标（或按 `Ctrl+Shift+X`）
3. 点击顶部的 `...` 按钮，选择 **从 VSIX 安装...**
4. 选择下载的 `.vsix` 文件完成安装

### 首次配置

1. 安装完成后，插件会自动激活
2. 点击 VS Code 右下角的状态栏图标，打开设置界面
3. 配置您要使用的 AI 模型 API 密钥
4. 根据需要配置其他参数（如编译命令、代码补全设置等）

## 2. 核心功能操作指南

### 2.1 多模型 AI 聊天

**访问方式：**

- 点击 VS Code 左侧边栏的聊天图标，打开聊天窗口
- 或使用命令面板（`Ctrl+Shift+P`）输入 `LLMA: 打开聊天窗口`

**聊天模式：**

- **聊天模式**：适合回答编程问题、解释代码
- **Agent 模式**：适合自动生成/修改文件、执行终端命令

**切换模式：**

- 使用快捷键 `Ctrl+Shift+Q`
- 或在聊天界面顶部点击模式切换按钮

**发送消息：**

1. 在输入框中输入您的问题或指令
2. 按 `Enter` 发送消息（`Shift+Enter` 换行）
3. 等待 AI 响应（支持流式输出）

**上传图片（多模态支持）：**

- 点击输入框右侧的图片图标
- 选择本地图片文件
- 图片会作为上下文发送给 AI（需模型支持）

### 2.2 智能上下文感知

📄**自动上下文：**

- 插件会自动将当前打开的编辑器内容作为上下文
- 若有选中代码，会优先使用选中部分作为上下文

📝**手动添加文件：**

1. 在聊天界面中点击“添加文件到上下文”按钮
2. 在文件选择对话框中选择要添加的文件
3. 确认后，文件内容会作为上下文提供给 AI

### 2.3 🌐联网搜索

**配置步骤：**

1. 打开设置界面
2. 启用 `enableWebSearch` 选项
3. 输入 `serpApiKey`（需在 [SerpApi 官网](https://serpapi.com) 注册获取）
4. 选择 `webSearchEngine`（如 google、bing、baidu 等）

**使用方法：**

- 在聊天界面中，勾选“使用联网搜索”选项
- 发送消息后，AI 会结合搜索结果回答问题

### 2.4 行内代码预测（Ghost Text）

**启用方法：**

- 在设置中开启 `enableAutoCompletion` 选项
- 可调整 `requestDelay` 设置延迟触发时间

**使用方法：**

- 正常编写代码时，插件会自动预测并显示补全建议
- 按 `Tab` 键接受补全建议
- 按 `Alt+\` 手动触发补全

### 2.5 一键编译与运行

**支持的语言：**

- C/C++、Java、Python、JavaScript、TypeScript、Rust、Go、仓颉等

**编译操作：**

1. 打开要编译的文件
2. 使用快捷键 `Ctrl+Shift+B`
3. 编译结果会显示在聊天窗口中

**运行操作：**

- 编译成功后，点击聊天窗口中的“运行”按钮
- 程序会在独立终端中执行

### 2.6 🤖Agent 模式工具使用

**文件操作工具：**

1. **创建/覆盖文件：**
   - AI 会生成 `> FILE: <filepath>` 指令
   - 点击“应用更改”按钮创建/覆盖文件

2. **读取文件：**
   - AI 会生成 `> READ: <filepath>` 指令
   - 插件会读取文件内容并提供给 AI

3. **修改文件：**
   - AI 会生成各种修改指令（如 `> REPLACE`、`> EDIT_FUNCTION` 等）
   - 点击“应用更改”按钮执行修改
   - 插件会自动备份原文件，可随时撤销更改

**命令执行工具：**

1. AI 会生成 `> RUN: <command>` 指令
2. 点击“执行命令”按钮
3. 确认命令执行（可在设置中配置为自动允许）
4. 执行结果会返回给 AI 继续推理

### 2.7 任务规划与执行

**自动任务规划：**

- 对于复杂任务，AI 会自动分解为可执行的步骤序列
- 显示任务风险等级和执行进度
- 支持步骤间的依赖关系管理

**查看任务计划：**

- 在 Agent 模式下，AI 会在执行前显示任务计划
- 您可以查看并确认计划后再执行

### 2.8 图形化设置界面

**访问方式：**

- 点击 VS Code 右下角的状态栏图标
- 或在聊天界面中点击齿轮图标⚙️

**配置选项：**

- **模型设置**：选择 AI 模型、配置 API 密钥
- **补全设置**：调整代码补全参数
- **编译设置**：配置各语言的编译命令
- **Agent 设置**：调整 Agent 行为和权限
- **联网搜索**：配置搜索功能

## 4. 高级功能使用

### 4.1 会话管理

- **保存会话**：点击聊天界面顶部的“保存会话”按钮
- **加载会话**：点击“会话列表”按钮，选择要加载的会话
- **删除会话**：在会话列表中选择会话，点击删除按钮

### 4.2 文件操作高级技巧

- **精确编辑**：使用 `> REPLACE` 指令进行小范围精确修改
- **函数/类编辑**：使用 `> EDIT_FUNCTION` 或 `> EDIT_CLASS` 指令修改整个函数或类
- **批量操作**：使用 `> MULTI_FILE` 指令同时创建或修改多个文件

### 4.3 编译配置自定义

- 在设置中修改 `llma.compilation.compilers` 配置
- 支持使用占位符，如 `{file}`、`{executable}` 等
- 可自定义各语言的编译命令

## 5. 常见问题与解决方案

### 5.1 模型连接问题

- **API 密钥错误**：检查设置中的 API 密钥是否正确
- **网络连接问题**：确保网络连接正常，特别是使用在线模型时
- **本地模型未启动**：使用本地模型时，确保本地服务已启动

### 5.2 编译失败问题

- **编译器未安装**：确保已安装对应语言的编译器
- **编译命令错误**：检查设置中的编译命令配置
- **文件路径问题**：确保文件路径正确，特别是包含空格的路径

### 5.3 文件修改冲突

- **文件已被外部修改**：插件会备份原文件，可使用“撤销更改”功能恢复
- **权限问题**：确保有文件修改权限

### 5.4 性能问题

- **AI 响应缓慢**：尝试调整模型参数，如减小 `maxTokens`
- **内存占用高**：关闭不必要的功能，如禁用联网搜索

## 6. 最佳实践

1. **合理使用模式：**
   - 简单问题使用聊天模式
   - 文件操作和命令执行使用 Agent 模式

2. **提供清晰的指令：**
   - 明确说明您的需求
   - 提供足够的上下文信息
   - 对于复杂任务，分步说明

3. **利用上下文功能：**
   - 选中相关代码片段
   - 添加相关文件作为参考
   - 对于大型项目，提供项目结构说明

4. **安全使用命令执行：**
   - 仔细检查 AI 生成的命令
   - 避免执行危险命令
   - 在生产环境中谨慎使用

5. **定期保存会话：**
   - 对于重要的对话，及时保存会话
   - 整理和管理会话历史
